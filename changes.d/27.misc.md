Drop Python 3.10 (now 3.11–3.14) and pin the local interpreter to the exact patch `3.14.6`. CPython 3.14.0 has a numpy crash (a `bool |= float32-array == scalar` compare segfaults — exactly the idiom xarray uses to decode fill values), fixed in later 3.14 patches; a bare `3.14` pin floated to the broken 3.14.0. Fetching 3.14.6 needs uv ≥ 0.11, so the CI `setup-uv` pin is bumped too. Dropping 3.10 also lets the `xarray` cap move off the stale `<2025` (3.10 was xarray's blocker — its last 3.10 release is 2025.6.1), so `xarray` now resolves to the current 2026.x line. The other stale calendar/major caps are lifted alongside it — `pandas` (`<3` → `<4`, now 3.0.3) and the `dask`/`distributed` extra pins (`<2025` → `<2027`, now 2026.x).
